业务实现的总结：
   所有功能实现和业务需求都是需要在方法中进行实现
     *所以方法是业务实现的基本构造
     *一想到方法和业务逻辑分析，首先想到的我们想要的结果，也就是返回类型
     *页面如何展示，取决于返回甚么结果，并展示上去。
     *当然也有无需返回结果的，也有可以相应的返回状态码。
     *结果的获取，必然是查询，所以我们先进行查询，然后对结果进一步封装，也就是进行改造。改造成自己需要的返回类型。
     *同时获取结果需要传递相应的参数，对应的参数也能设置成一个我们想要的数据类型，方便使用。
     *当请求参数和返回结果确定了，中间这一步就是逻辑处理，如何利用请求参数，去查询获取结果，再如何把结果封装成自己的返回类型。这其中也就是数据转换。
     *其中出错，如何处理，其中可能出现多种情况，如何处理。这其中就是处理错误。

业务开发的注意事项：
        1.前后端参数的不统一会导致取不出来数据，可以建立专门的实体类进行参数的传递。
        2.接口文档的规范性 Restful接口规范
        3.文件命名规范 属性注意相关大小写
        4.记住前端的数据不是实时的，也就是不是最正确，需要自己进行查询最新的数据进行判断
