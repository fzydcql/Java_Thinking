错误分析：mybatis流程
1.先判断业务逻辑，业务逻辑没有问题
2.检查数据类型对应
3.检查xml，id对应等一些需要对应的参数。
4.再检查sql，用可视化工具直接判断sql语句是否出错
5.若sql正确则返回前面步骤查询
6.若sql不正确则修改sql，若sql正确则百度错误可能原因。

错误分析：数组越界异常
list没有值但是直接取值了，导致数组越界异常
对数组进行判断，可以直接判断它的长度，而不是为空字符。


错误分析： 数据库连接不上，导致SpringBoot启动自动销毁
1.一般会是mapper.xml出了问题
   .xml常见问题：id不对应，语句错误等。
2.service问题
3.mapper对应问题

错误分析：一些细节没有写好，明明逻辑正确但是却执行错误，就应该去查询代码中的相关细节方面。
1.语句问题
2.代码位子问题


数据类型导致的存储显示问题，小数是否会被完整存储，类型的不同会导致存储的数据变化，所以需要注意存储的数据类型，所表示的数据
